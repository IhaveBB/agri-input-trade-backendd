# 农资分类模块改动说明

## 一、改动概述

根据《农资分类模块设计》文档，对现有分类功能进行了以下主要改动：

### 1.1 数据结构扩展

原有 `Category` 实体只包含基础字段，现新增以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| parent_id | Long | 父分类ID，顶级分类为0 |
| level | Integer | 分类层级（1-一级，2-二级，3-三级，4-四级） |
| sort_order | Integer | 排序字段，数字越小越靠前 |
| status | Integer | 状态：0-禁用，1-启用 |
| is_custom | Integer | 是否商家自定义：0-系统预置，1-商家自定义 |
| create_user_id | Long | 创建用户ID |

### 1.2 新增功能

1. **多级分类支持**：支持四级分类结构（种子→粮食种子→谷物类→水稻）
2. **分类树形结构**：新增获取树形分类的接口
3. **商家自定义分类**：商家可申请新增自定义分类
4. **分类初始化数据**：系统启动时自动初始化预置分类
5. **作物数据统一**：作物数据存储在种子分类的四级分类中，农药/肥料选择适用作物时使用

## 二、数据库表结构

请在数据库中执行以下SQL创建/更新表结构：

```sql
CREATE TABLE IF NOT EXISTS `category` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '分类ID',
  `name` VARCHAR(100) NOT NULL COMMENT '分类名称',
  `icon` VARCHAR(255) DEFAULT NULL COMMENT '分类图标',
  `description` VARCHAR(500) DEFAULT NULL COMMENT '分类描述',
  `parent_id` BIGINT DEFAULT 0 COMMENT '父分类ID，顶级分类为0',
  `level` INT NOT NULL DEFAULT 1 COMMENT '分类层级：1-一级分类，2-二级分类，3-三级分类，4-四级分类',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序字段，数字越小越靠前',
  `status` INT NOT NULL DEFAULT 1 COMMENT '分类状态：0-禁用，1-启用',
  `is_custom` INT NOT NULL DEFAULT 0 COMMENT '是否为商家自定义分类：0-系统预置，1-商家自定义',
  `create_user_id` BIGINT DEFAULT NULL COMMENT '创建用户ID',
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_parent_id` (`parent_id`),
  KEY `idx_level` (`level`),
  KEY `idx_status` (`status`),
  KEY `idx_is_custom` (`is_custom`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='商品分类表';
```

**注意**：如果表已存在，需要执行以下ALTER语句添加新字段：

```sql
ALTER TABLE `category`
ADD COLUMN `parent_id` BIGINT DEFAULT 0 COMMENT '父分类ID，顶级分类为0' AFTER `description`,
ADD COLUMN `level` INT NOT NULL DEFAULT 1 COMMENT '分类层级' AFTER `parent_id`,
ADD COLUMN `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序字段' AFTER `level`,
ADD COLUMN `status` INT NOT NULL DEFAULT 1 COMMENT '分类状态：0-禁用，1-启用' AFTER `sort_order`,
ADD COLUMN `is_custom` INT NOT NULL DEFAULT 0 COMMENT '是否为商家自定义分类' AFTER `status`,
ADD COLUMN `create_user_id` BIGINT DEFAULT NULL COMMENT '创建用户ID' AFTER `is_custom`;

-- 添加索引
ALTER TABLE `category` ADD INDEX `idx_parent_id` (`parent_id`);
ALTER TABLE `category` ADD INDEX `idx_level` (`level`);
ALTER TABLE `category` ADD INDEX `idx_status` (`status`);
ALTER TABLE `category` ADD INDEX `idx_is_custom` (`is_custom`);
```

### 3.5 创建分类（管理员）

**接口**: `POST /api/category`

**说明**: 创建新分类（需管理员权限）

**请求头**: 需要携带登录token

**请求体**:
```json
{
  "name": "新分类名称",
  "icon": "图标URL",
  "description": "分类描述",
  "parentId": 0,  // 顶级分类填0
  "sortOrder": 0,
  "status": 1
}
```

**字段说明**:
| 字段名 | 必填 | 说明 |
|--------|------|------|
| name | 是 | 分类名称 |
| parentId | 否 | 父分类ID，顶级分类填0或不填，默认0 |
| sortOrder | 否 | 排序号，默认0 |
| status | 否 | 状态，默认1启用 |

---

### 3.6 商家申请新增自定义分类

**接口**: `POST /api/category/custom`

**说明**: 商家用户申请新增自定义分类

**请求头**: 需要携带商家登录token

**请求体**:
```json
{
  "name": "商家自定义分类",
  "parentId": 1,  // 必须选择已有父分类
  "description": "自定义分类描述"
}
```

**规则说明**:
1. 必须选择父分类（不能创建一级分类）
2. 同级分类下不能有同名
3. 名称不能与现有分类相似（如已有"小麦"，不能申请"高产小麦"）
4. 最多支持三级分类，不能在三级分类下再添加
5. 自定义分类默认启用

### 3.8 删除分类

**接口**: `DELETE /api/category/{id}`

**说明**: 删除分类

**规则**:
1. 如果存在子分类，无法删除
2. 如果存在关联商品，无法删除

```javascript
  if (result.code === 200) {
    return result.data;
```

### 4.4 商品列表筛选

```javascript
// 分页获取分类列表
async function loadCategories(page = 1, size = 10, level = null) {
  let url = `/api/category/page?currentPage=${page}&size=${size}`;
  if (level) {
    url += `&level=${level}`;
  }
  const response = await fetch(url);
  const result = await response.json();
  return result.data;
}
```

### 4.5 商家申请自定义分类

```javascript
// 商家申请新增分类
async function applyCustomCategory(categoryData) {
  const response = await fetch('/api/category/custom', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`  // 需要携带token
    },
    body: JSON.stringify(categoryData)
  });
  const result = await response.json();
  return result;
}

// 使用示例
await applyCustomCategory({
  name: '草莓种子',
  parentId: 9,  // 蔬菜种子
```

## 六、预置分类数据

系统启动时会自动初始化以下预置分类：

### 一级分类（7个）
- 种子、农药、肥料、饲料、兽药、农膜、农机

### 二级分类（部分）
- 种子：粮食种子、蔬菜种子、水果种子、花卉种子、林木种子、中药材种子
- 农药：杀虫剂、杀螨剂、杀菌剂、除草剂、植物生长调节剂、杀线虫剂、杀鼠剂
- 肥料：单质肥料、复合肥料、有机肥料、生物肥料、微量元素肥料、水溶肥料
- 饲料：粗饲料、青绿饲料、青贮饲料、能量饲料、蛋白质补充料、矿物质饲料、维生素饲料
- 兽药：抗菌药物、抗病毒药物、抗寄生虫药物
- 农膜：地膜、棚膜、遮阳网、防虫网
- 农机：生产机械、运输机械、加工机械

### 三级分类（部分）
- 种子→粮食种子：谷物类、杂粮类、豆类作物
- 种子→蔬菜种子：叶菜类、根茎类、茄果类、瓜类、豆类蔬菜、菌类、芽苗类
- 肥料→单质肥料：氮肥、磷肥、钾肥

### 四级分类（具体作物品种，仅种子分类）
- 种子→粮食种子→谷物类：水稻、小麦、玉米、谷子
- 种子→粮食种子→杂粮类：高粱、荞麦
- 种子→粮食种子→豆类作物：大豆、豌豆
- 种子→蔬菜种子→叶菜类：白菜、生菜、菠菜
- 种子→蔬菜种子→根茎类：胡萝卜、萝卜、土豆
- 种子→蔬菜种子→茄果类：番茄、茄子、辣椒
- 种子→蔬菜种子→瓜类：黄瓜、西瓜、南瓜

# 推荐功能增强 - 地理位置权重

## 需求说明

在基于用户的协同过滤推荐算法中加入位置权重，优先推荐同区域用户的购买商品。

- 同市：权重 × 1.5
- 同省：权重 × 1.2
- 同区域（华北/华南等）：权重 × 1.1
- 不同区域：权重 × 1.0

## 数据库变更

### user 表新增字段

```sql
ALTER TABLE user ADD COLUMN location VARCHAR(255) COMMENT '位置信息';
```

location 字段格式：`省-市` 或 `省-市-区`

示例：
- `广东省-深圳市`
- `广东省-深圳市-南山区`

## 算法说明

### 1. 评分矩阵构建

用户-商品评分矩阵：
- 购买行为：2.0分
- 收藏行为：1.0分

```java
Map<Long, Map<Long, Double>> userRatingMatrix;
// 外层Key: 用户ID
// 内层Key: 商品ID
// Value: 评分
```

### 2. 皮尔逊相关系数

**为什么用皮尔逊相关系数**：
- 余弦相似度只关注方向，不关注绝对值差异
- 皮尔逊相关系数**消除用户评分偏好的影响**

**举例**：
- 用户A给所有商品打4-5分（偏高）
- 用户B给所有商品打1-2分（偏低）
- 两人品味其实很相似（都买相同的农药），但余弦相似度很低
- 皮尔逊相关系数能正确识别这种趋势一致

公式：
```
r = Σ((xi - x̄)(yi - ȳ)) / (√Σ(xi - x̄)² × √Σ(yi - ȳ)²)
```

其中：
- xi, yi 是用户对商品i的评分
- x̄, ȳ 是用户x和y的平均评分

**最小共同商品数**：只有共同商品数 >= 2 时才计算相似度，避免偶然相似

### 3. 位置权重

最终相似度 = 行为相似度 × 位置权重

| 位置关系 | 权重 |
|---------|------|
| 同市 | 1.5 |
| 同省 | 1.2 |
| 同区域 | 1.1 |
| 不同区域 | 1.0 |

### 4. 推荐分数计算

使用皮尔逊相关系数后的预测评分公式：
```
预测评分 = 用户平均分 + Σ(相似度 × (商品评分 - 用户平均分)) / Σ|相似度|
```

这样可以：
- 消除用户评分偏好的影响
- 预测该用户对商品的真实喜好程度

### 5. 相似度阈值

| 用户行为数 | 阈值 |
|-----------|------|
| < 3 (新用户) | 0.2 |
| 3-10 (普通用户) | 0.3 |
| > 10 (活跃用户) | 0.4 |

## 设计规范遵循

1. **阿里巴巴开发手册**：
   - 使用常量类替代魔法数字
   - DTO 前后端分离
   - 接口添加 Swagger 注解

2. **设计模式**：
   - 配置属性类：解耦硬编码
   - 常量类：统一管理配置
   - DTO：数据传输对象隔离
   - 缓存优化：减少重复计算

3. **可扩展性**：
   - 位置权重可通过配置文件调整
   - 省份-区域映射可轻松扩展
   - 行为权重可通过常量类调整

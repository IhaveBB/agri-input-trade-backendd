# 一、基础业务流程设计

## 1.1 商品发布总体流程

商家在发布商品时：

1. 先选择一级分类
2. 根据分类动态展示不同扩展信息模块
3. 填写基础信息
4. 填写推荐过滤相关信息
5. 提交保存

------

## 1.2 分类驱动的动态信息模块

不同一级分类，触发不同扩展表单：

| 一级分类 | 需填写推荐维度 |
| -------- | -------------- |
| 种子     | 区域 + 季节    |
| 农药     | 适用作物       |
| 化肥     | 适用作物       |
| 其他农资 | 仅填写基础信息 |

------

## 1.3 设计原则

- 推荐相关字段仅在需要的分类中出现
- 非相关分类不展示无意义字段
- 保证数据最小化原则

------

# 二、数据库设计

------

## 2.1 商品主表（product）

仅存储通用静态信息：

- 名称
- 分类
- 品牌
- 价格
- 库存
- 销量
- 状态

不存：

- 推荐权重
- 用户相关数据

------

## 2.2 商品适用作物表（适用于农药/肥料）

```text
product_crop
------------
id
product_id
category_id    -- 对应category表的种子分类四级分类
```

说明：

- 作物数据存储在分类表（category）中
- 使用种子分类的四级分类作为具体作物（如水稻、小麦、玉米等）
- 支持选择三级分类或四级具体作物
- 用于用户作物匹配推荐

------

## 2.4 区域表（region）

| 字段       | 说明     |
| ---------- | -------- |
| id         | 区域ID   |
| name       | 区域名称 |
| sort_order | 排序     |
| status     | 状态     |

示例数据：

- 华北
- 华东
- 华南
- 东北
- 西北
- 西南
- 全国

说明：

- 用于种子适配区域建模
- 支持扩展与排序
- 支持后台管理

------

## 2.5 季节表（season）

| 字段 | 说明     |
| ---- | -------- |
| id   | 季节ID   |
| name | 季节名称 |

示例：

- 春
- 夏
- 秋
- 冬
- 全年

说明：

- 表示种子适种时间维度

------

## 2.6 商品适用区域-季节映射表

```text
product_region_season
----------------------
id
product_id
region_id
season_id
```

说明：

该表表示三元关系：

> 商品 × 区域 × 季节

每条记录代表一个“适用关系对”。

例如：

| 商品 | 区域 | 季节 |
| ---- | ---- | ---- |
| A    | 华北 | 春   |
| A    | 华北 | 秋   |
| A    | 华南 | 全年 |

说明：

- 不是存集合
- 每个 (区域, 季节) 组合独立存储
- 满足关系型数据库第一范式

# 三、前端功能设计

# 3.1 商品发布页面改造

## 3.1.1 一级分类切换逻辑

当商家选择一级分类时：

前端根据分类动态渲染不同模块：

- 选择“种子” → 展示区域-季节配置
- 选择“农药/化肥” → 展示作物树选择
- 其他分类 → 不展示推荐过滤维度模块

实现方式：

- 监听分类变化
- 控制条件渲染模块

------

# 3.2 种子：区域-季节适用范围设计

## UI结构

```
适种区域配置

[ + 添加区域 ]

区域：华北
适宜季节：☐ 春 ☐ 秋

区域：华南
适宜季节：☐ 全年
```

## 交互规则

1. 可添加多个区域块
2. 每个区域块必须选择至少一个季节
3. 选择“全国”时：
   - 禁用其他区域选择
4. 季节允许多选
5. 删除区域块自动删除对应关系

## 数据提交格式

前端提交结构：

```json
[
  {
    "region_id": 1,
    "season_ids": [1,3]
  }
]
```

后端拆分为多条记录写入 product_region_season。

------

# 3.3 农药、化肥：适用作物选择

## UI组件

树形多选组件：

```
粮食类
 ├── 水稻
 ├── 小麦
水果类
 ├── 苹果
```

支持：

- 父级选择
- 子级选择
- 多选

------

## 数据存储规则

选择结果：

- 直接存入 product_crop 表
- 不做扁平化合并

# 四、不同分类基础信息字段说明

A.1 种子 可标明作物种类与种子类别、品种名称、生产商或进口商或分装单位名称与地址、质量指标、品种说 明(品种主要性状、主要栽培措施、使用条件的说明)、净含量、生产年月、农作物种子经营许可证编号、警 示标志、是否转基因等。 A.2 农药 可标明农药登记证号、农药生产许可证号、有效成分含量、剂型、使用范围、剂量和使用方法、净含 量、产品质量保证期、毒性等级、注意事项、贮存和运输方法、生产者的名称和地址等。 A.3 肥料 可标明等级、净含量、养分含量、其他添加物含量、生产者或经销者的名称与地址、生产日期、警示说 明等。 A.4 饲料 可标明产品成分分析保证值、原料组成、净含量、生产日期、保质期、生产者或经销者的名称与地址、 注意事项等。 A.5 兽药 可标明有效成分、药理作用、适用症状、用法用量、休药期、不良反应、保质期及注意事项等。 A.6 农膜 可标明适用范围、类别、称重厚度、宽度、参考长度等。 A.7 农业机具及配件 可标明用途、产品结构参数、材质等。

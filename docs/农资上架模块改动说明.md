# 农资上架模块改动说明

## 一、改动概述

根据《农资上架部分》设计文档，对现有商品模块进行了以下主要改动：

### 1.1 新增功能

1. **分类驱动的动态扩展字段**：不同一级分类显示不同的扩展信息模块
2. **适用作物选择**：农药、肥料可选择适用的作物
3. **区域-季节配置**：种子类商品可配置适种区域和季节
4. **JSON格式存储扩展属性**：使用JSON字段存储动态扩展属性，便于扩展

### 1.2 新增数据库表

- `region` - 区域表
- `season` - 季节表
- `product_crop` - 商品适用作物关联表（作物使用category表的种子分类）
- `product_region_season` - 商品适用区域-季节关联表

> **注意**：作物数据直接使用分类表（category）中种子分类的四级分类数据，无需单独建作物表。

### 1.3 Product表扩展

- 新增 `extra_attributes` JSON字段存储动态扩展属性
- 新增 `deleted` 字段支持逻辑删除

## 二、数据库表结构

请执行 `docs/sql/product_ext_tables.sql` 创建表结构：

```sql
-- 区域表
CREATE TABLE `region` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `name` VARCHAR(50) NOT NULL,
  `sort_order` INT DEFAULT 0,
  `status` INT DEFAULT 1,
  ...
);

-- 季节表
CREATE TABLE `season` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `name` VARCHAR(20) NOT NULL,
  ...
);

-- 商品适用作物关联表（作物使用category表的种子分类）
CREATE TABLE `product_crop` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `product_id` BIGINT NOT NULL,
  `category_id` BIGINT NOT NULL COMMENT '作物分类ID（对应category表的种子分类四级分类）'
);

-- 商品适用区域-季节关联表
CREATE TABLE `product_region_season` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `product_id` BIGINT NOT NULL,
  `region_id` BIGINT NOT NULL,
  `season_id` BIGINT NOT NULL
);
```

## 三、接口文档

### 3.1 商品接口（扩展版）

#### 3.1.1 创建商品（含扩展信息）

**接口**: `POST /api/product/ext`

**说明**: 创建商品，支持扩展字段和推荐维度

**请求头**: 需要携带登录token

**请求体**:
```json
{
  "name": "高产水稻种子",
  "description": "优质水稻种子",
  "price": 88.00,
  "stock": 1000,
  "categoryId": 8,
  "imageUrl": "https://xxx.com/img.jpg",
  "placeOfOrigin": "湖南",

  "extraAttributes": {
    "seedType": 2,
    "varietyName": "超级稻",
    "manufacturer": "隆平高科",
    "netContent": "500g",
    "isGmo": 0
  },

  "cropIds": [1, 2, 3],

  "regionSeasonConfigs": [
    {
      "regionId": 1,
      "seasonIds": [1, 3]
    },
    {
      "regionId": 2,
      "seasonIds": [1, 2, 5]
    }
  ]
}
```

**字段说明**:
| 字段名 | 类型 | 说明 |
|--------|------|------|
| name | String | 商品名称 |
| description | String | 商品描述 |
| price | BigDecimal | 价格 |
| stock | Integer | 库存 |
| categoryId | Long | 分类ID |
| imageUrl | String | 图片URL |
| placeOfOrigin | String | 产地 |
| extraAttributes | Map | 扩展属性Map（JSON格式） |
| cropIds | List<Long> | 适用作物ID列表（农药/肥料用） |
| regionSeasonConfigs | List | 区域-季节配置（种子用） |

**extraAttributes扩展属性说明**:

不同分类对应不同的属性Key，详见接口 `GET /api/product/ext/fields`

#### 3.1.2 更新商品（含扩展信息）

**接口**: `PUT /api/product/ext/{id}`

**说明**: 更新商品扩展信息

**请求体**: 同创建

**前端使用示例**:
```javascript
// 根据字段配置动态渲染表单
function renderForm(fields) {
  return fields.map(field => {
    if (field.type === 'text') {
      return `<input v-model="extraAttributes.${field.key}" placeholder="${field.label}" />`;
    } else if (field.type === 'textarea') {
      return `<textarea v-model="extraAttributes.${field.key}"></textarea>`;
    } else if (field.type === 'select') {
      const options = field.options.map(o =>
        `<option value="${o.value}">${o.label}</option>`
      ).join('');
      return `<select v-model="extraAttributes.${field.key}">${options}</select>`;
    }
  }).join('');
}
```

## 四、前端使用说明

### 4.1 商品发布流程

```
1. 选择一级分类
       ↓
2. 根据分类动态显示扩展字段
       ↓
3. 填写基础信息 + 扩展信息
       ↓
4. （种子）配置区域-季节
       ↓
5. （农药/肥料）选择适用作物
       ↓
6. 提交保存
```

#### 4.3.4 提交商品数据

```javascript
async function submitProduct(productData) {
  const response = await fetch('/api/product/ext', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({
      name: productData.name,
      price: productData.price,
      categoryId: productData.categoryId,
      // 基础字段...

      // 扩展属性（Map格式）
      extraAttributes: {
        seedType: productData.seedType,
        varietyName: productData.varietyName,
        manufacturer: productData.manufacturer,
        isGmo: productData.isGmo,
        // 其他扩展属性...
      },

      // 农药/肥料 - 适用作物
      cropIds: productData.selectedCrops.flat(),

      // 种子 - 区域-季节配置
      regionSeasonConfigs: productData.regionBlocks.map(block => ({
        regionId: block.regionId,
        seasonIds: block.seasonIds
      }))
    })
  });

  const result = await response.json();
  return result;
}
```

### 4.5 扩展属性动态渲染示例

```javascript
// 根据字段配置动态渲染表单
function renderExtFields(extFields) {
  return extFields.map(field => {
    if (field.type === 'text') {
      return `<input v-model="extraAttributes.${field.key}" placeholder="${field.label}" />`;
    } else if (field.type === 'textarea') {
      return `<textarea v-model="extraAttributes.${field.key}"></textarea>`;
    } else if (field.type === 'select') {
      const options = field.options.map(opt => {
        const [value, label] = opt.split('-');
        return `<option value="${value}">${label}</option>`;
      }).join('');
      return `<select v-model="extraAttributes.${field.key}">${options}</select>`;
    }
  }).join('');
}
```

## 五、预置数据

### 5.1 区域数据
华北、华东、华南、华中、东北、西北、西南、全国

### 5.2 季节数据
春、夏、秋、冬、全年

### 5.3 作物数据
> 作物数据直接使用分类表中种子分类的数据。
> 分类表支持四级分类：一级（种子）→ 二级（粮食种子/蔬菜种子等）→ 三级（谷物类/叶菜类等）→ 四级（具体作物品种）

分类数据示例（种子分类）：
- 种子
  - 粮食种子
    - 谷物类 → 水稻、小麦、玉米、谷子
    - 杂粮类 → 高粱、荞麦
    - 豆类作物 → 大豆、豌豆
  - 蔬菜种子
    - 叶菜类 → 白菜、生菜、菠菜
    - 根茎类 → 胡萝卜、萝卜、土豆
    - 茄果类 → 番茄、茄子、辣椒
    - 瓜类 → 黄瓜、西瓜、南瓜
  - 水果种子
  - 花卉种子
  - 林木种子
  - 中药材种子

农药/肥料选择适用作物时，可选择三级分类或四级具体作物。
